# AI互动视觉小说引擎 — 技术架构与实现方案

## 1. 项目定位与验证目标

本项目的核心目标是构建一个 **AI驱动的互动视觉小说引擎**，其中角色拥有多维记忆、自主决策能力和动态人格演化。

**关键验证点：Agent记忆系统的效果迭代**
- 记忆的分层召回是否能保证角色行为一致性
- 5W1H推理在记忆上下文变化时是否产生合理的行为涌现
- 记忆反思机制（Consolidation）是否有效驱动人格演化

为支撑这一验证目标，需要对所有Agent调用实现**完整的可观测性**（Observability），包括每次LLM推理的输入prompt、输出、token消耗、延迟、tool调用链路等。

## 2. 技术栈选型

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **Agent框架** | [Mastra](https://mastra.ai/) | TypeScript原生Agent框架，内置Observability/Memory/Workflow/MCP |
| **游戏引擎** | Pixi'VN + NQTR扩展 | 解耦渲染与UI，程序化控制叙事步骤，内置时间系统 |
| **前端UI** | React + Zustand | Pixi'VN官方支持React，Zustand管理角色记忆状态 |
| **结构化输出** | Zod | 5W1H目标schema验证，确保LLM输出类型安全 |
| **向量存储** | LibSQL / PGVector | Mastra原生支持，用于情节记忆的RAG检索 |
| **可观测性** | Mastra Observability + OpenTelemetry | 内置trace导出，支持Langfuse/Datadog等外部平台 |
| **MCP** | @mastra/mcp | 通过MCP协议对接外部工具和资源 |

### 2.1 为什么选择 Mastra 而非 Vercel AI SDK

PRD中原方案使用Vercel AI SDK，但本项目需要验证Agent记忆系统，Mastra在以下方面有显著优势：

1. **内置Memory系统**：支持Working Memory（短期）、Semantic Recall（语义检索）、Observational Memory（自动压缩长对话），直接对应PRD中的多维记忆需求
2. **内置Observability**：自动采集Agent决策路径、tool调用、token消耗，无需额外集成
3. **Workflow编排**：用`createWorkflow()`编排多步Agent调用（5W1H → GOAP → Director），比手动管理Promise链更可靠
4. **MCP支持**：通过MCPClient/MCPServer暴露和消费工具，Agent能力可插拔扩展
5. **Multi-Agent**：Supervisor模式天然支持PRD中的"认知Agent + 导演Agent"分离架构

Mastra底层仍基于AI SDK，不会损失流式输出等核心能力。

## 5. 多维记忆架构 × 兼容OpenClaw ContextEngine协议

### 5.0 设计目标：ContextEngine兼容的插件化记忆

OpenClaw v2026.3.7 引入了 **ContextEngine** 插件协议，将上下文管理（组装、压缩、子Agent隔离）从框架核心解耦为可替换的插件槽位。本项目的记忆系统采用**同一套生命周期接口**，这样：

1. 我们的记忆引擎可以**直接作为OpenClaw的ContextEngine插件**运行（验证通用性）
2. 未来可以接入OpenClaw生态中的第三方记忆插件（如Lossless Claw的DAG压缩）
3. Mastra Memory作为底层存储实现，ContextEngine接口作为上层编排协议

### 5.1 ContextEngine 7个生命周期钩子 × 本项目映射

OpenClaw ContextEngine定义了7个生命周期钩子。我们逐一映射到互动小说引擎的场景：

| ContextEngine钩子 | 职责 | 本项目对应行为 |
|---|---|---|
| **`bootstrap`** | 引擎启动初始化 | 加载角色Core Persona JSON、初始化Mastra Memory连接、预热向量索引 |
| **`ingest`** | 新信息/消息进入上下文 | 每次事件发生时写入情节记忆：世界事件广播、对话记录、行动结果、玩家输入 |
| **`assemble`** | 组装最终发送给LLM的上下文 | **核心循环步骤①**：从Zustand取人格/目标 + Semantic Recall检索情节记忆 + 注入世界状态 → 拼装成System Prompt |
| **`compact`** | 压缩/裁剪上下文 | Reflection Agent的触发入口：分析近期记忆、生成人格修正补丁、压缩冗余情节为摘要 |
| **`afterTurn`** | 一轮对话完成后处理 | **核心循环步骤⑨**：更新短期目标状态、写入完成事件、推进时间线、评估是否触发compact |
| **`prepareSubagentSpawn`** | 子Agent生成前准备上下文 | Director Agent / Reflection Agent被Supervisor调起前，裁剪并注入角色专属的上下文切片 |
| **`onSubagentEnded`** | 子Agent结束后回收上下文 | Director输出的叙事文本回写到情节记忆；Reflection的人格补丁合并到Zustand Store |

### 5.2 与OpenClaw ContextEngine的差异分析

逐方法对比我们原方案与OpenClaw真实接口的差距：

| 方法 | OpenClaw真实签名 | 我们原方案 | 差距 |
|------|-----------------|-----------|------|
| **方法模式** | 统一 `params` 对象模式 | 位置参数混用 | ❌ 需改为params对象 |
| **`bootstrap?`** | `(params: {sessionId, sessionFile}) → BootstrapResult` | `() → void` | ❌ 缺sessionId/返回值 |
| **`ingest`** | `(params: {sessionId, message: AgentMessage, isHeartbeat?}) → IngestResult` | `(event: GameEvent) → IngestResult` | ❌ 参数结构不同 |
| **`ingestBatch?`** | `(params: {sessionId, messages[], isHeartbeat?}) → IngestBatchResult` | 无 | ❌ 完全缺失 |
| **`assemble`** | `(params: {sessionId, messages[], tokenBudget?}) → AssembleResult` | `(input: {messages, characterId, worldState}) → AssembleResult` | ❌ 缺tokenBudget，多了自定义字段 |
| **`compact`** | `(params: {sessionId, sessionFile, tokenBudget?, force?, currentTokenCount?, compactionTarget?, customInstructions?, runtimeContext?})` | `(characterId: string)` | ❌ 严重缺失 |
| **`afterTurn?`** | `(params: {sessionId, sessionFile, messages[], prePromptMessageCount, autoCompactionSummary?, isHeartbeat?, tokenBudget?, runtimeContext?})` | `(turnResult: TurnResult)` | ❌ 缺关键字段 |
| **`prepareSubagentSpawn?`** | `(params: {parentSessionKey, childSessionKey, ttlMs?}) → SubagentSpawnPreparation \| undefined` | `(agentId, parentContext) → AssembleResult` | ❌ 键命名/返回值不同 |
| **`onSubagentEnded?`** | `(params: {childSessionKey, reason: SubagentEndReason})` | `(agentId, agentOutput)` | ❌ 缺reason枚举 |
| **`dispose?`** | `() → void` | 无 | ❌ 完全缺失 |
| **可选性** | 大部分方法为 `?` 可选 | 全部必选 | ❌ 需调整 |

### 5.3 修正后的接口：严格对齐 + 游戏扩展层

设计策略：**底层100%对齐OpenClaw接口签名**，游戏特有数据通过 `runtimeContext` 透传，不污染核心协议。

```typescript
// src/memory/context-engine.interface.ts
// 严格对齐OpenClaw ContextEngine协议（v2026.3.7）

// ─── 核心类型（与OpenClaw完全一致）───────────────────────────

export interface ContextEngineInfo {
  id: string;
  name: string;
  ownsCompaction: boolean;
}

export interface AgentMessage {
  role: 'system' | 'user' | 'assistant' | 'tool';
  content: string;
  // OpenClaw标准字段
  [key: string]: unknown;
}

export interface BootstrapResult {
  ok: boolean;
  restoredMessageCount?: number;
}

export interface IngestResult {
  ingested: boolean;
}

export interface IngestBatchResult {
  ingested: boolean;
  count: number;
}

export interface AssembleResult {
  messages: AgentMessage[];
  estimatedTokens: number;
  // ⬇ 游戏扩展：通过可选metadata透传，不破坏协议
  metadata?: {
    recalledMemories?: EpisodicMemory[];
    activeGoals?: Goal5W1H[];
    personaShifts?: PersonaShift[];
  };
}

export interface CompactResult {
  ok: boolean;
  compacted: boolean;
  summary?: string;
  // ⬇ 游戏扩展
  personaShift?: PersonaShift;
}

export interface SubagentSpawnPreparation {
  rollback?(): Promise<void>;
}

export type SubagentEndReason = 'completed' | 'aborted' | 'error' | 'timeout';

// ─── runtimeContext：游戏数据的透传通道 ─────────────────────

export interface ContextEngineRuntimeContext {
  // OpenClaw标准：调用方可注入任意运行时状态
  [key: string]: unknown;
}

/** 游戏引擎专用的runtimeContext扩展 */
export interface NovelRuntimeContext extends ContextEngineRuntimeContext {
  characterId: string;
  worldState: WorldState;
  currentGoals: Goal5W1H[];
  personaShifts: PersonaShift[];
}

// ─── 基础接口（与OpenClaw签名100%一致，可选方法保持?）──────

export interface IContextEngine {
  readonly info: ContextEngineInfo;

  bootstrap?(params: {
    sessionId: string;
    sessionFile: string;
  }): Promise<BootstrapResult>;

  ingest(params: {
    sessionId: string;
    message: AgentMessage;
    isHeartbeat?: boolean;
  }): Promise<IngestResult>;

  ingestBatch?(params: {
    sessionId: string;
    messages: AgentMessage[];
    isHeartbeat?: boolean;
  }): Promise<IngestBatchResult>;

  assemble(params: {
    sessionId: string;
    messages: AgentMessage[];
    tokenBudget?: number;
  }): Promise<AssembleResult>;

  compact(params: {
    sessionId: string;
    sessionFile: string;
    tokenBudget?: number;
    force?: boolean;
    currentTokenCount?: number;
    compactionTarget?: 'budget' | 'threshold';
    customInstructions?: string;
    runtimeContext?: ContextEngineRuntimeContext;
  }): Promise<CompactResult>;

  afterTurn?(params: {
    sessionId: string;
    sessionFile: string;
    messages: AgentMessage[];
    prePromptMessageCount: number;
    autoCompactionSummary?: string;
    isHeartbeat?: boolean;
    tokenBudget?: number;
    runtimeContext?: ContextEngineRuntimeContext;
  }): Promise<void>;

  prepareSubagentSpawn?(params: {
    parentSessionKey: string;
    childSessionKey: string;
    ttlMs?: number;
  }): Promise<SubagentSpawnPreparation | undefined>;

  onSubagentEnded?(params: {
    childSessionKey: string;
    reason: SubagentEndReason;
  }): Promise<void>;

  dispose?(): Promise<void>;
}

// ─── 全量接口：本项目使用，所有方法必选 ─────────────────────
// 互动小说引擎需要完整的生命周期控制，不允许任何钩子缺失。
// 实现类 NovelContextEngine 必须 implements IFullContextEngine。

export interface IFullContextEngine extends Required<IContextEngine> {
  // Required<T> 将所有 ? 可选方法变为必选
  // 额外约束：dispose 返回值也从 void 提升为确认性返回
}

// 类型验证：确保 IFullContextEngine 仍然兼容 IContextEngine
// （即 NovelContextEngine 既能注册到OpenClaw插件槽，也满足本项目的全量要求）
type _AssertCompatible = IFullContextEngine extends IContextEngine ? true : never;
```

### 5.4 sessionId映射策略

OpenClaw以 `sessionId` 标识一个对话会话。在互动小说中，每个角色就是一个独立的"会话"：

```typescript
// sessionId = `novel:${characterId}`
// sessionFile = `memory/${characterId}.session.json`
//
// 示例：
//   角色"莉娅" → sessionId: "novel:liya"
//   角色"卫兵队长" → sessionId: "novel:guard-captain"
//
// 好处：
// 1. 天然支持多角色并行（各自独立session）
// 2. prepareSubagentSpawn的parent/child键自然映射为
//    parent = "novel:liya"
//    child  = "novel:liya:director"（莉娅的导演Agent子会话）
```

### 5.5 基于Mastra Memory的ContextEngine实现

```typescript
// src/memory/novel-context-engine.ts

import { Memory } from '@mastra/memory';
import { LibSQLStore } from '@mastra/libsql';
import type {
  IContextEngine, AssembleResult, BootstrapResult,
  IngestResult, IngestBatchResult, CompactResult,
  SubagentSpawnPreparation, SubagentEndReason,
  NovelRuntimeContext, AgentMessage,
} from './context-engine.interface';

export class NovelContextEngine implements IFullContextEngine {
  readonly info = { id: 'novel-engine', name: 'AI Novel Memory Engine', ownsCompaction: true };

  private memory: Memory;
  private characterStore: CharacterMemoryStore;  // Zustand

  constructor() {
    this.memory = new Memory({
      storage: new LibSQLStore({ url: 'file:./game-memory.db' }),
      options: {
        semanticRecall: { topK: 5, messageRange: { before: 2, after: 2 } },
        workingMemory: { enabled: true },
        observationalMemory: { enabled: true },
      },
    });
  }

  // ─── 生命周期：初始化 ─────────────────────────────────────

  async bootstrap(params: { sessionId: string; sessionFile: string }): Promise<BootstrapResult> {
    const characterId = this.parseCharacterId(params.sessionId); // "novel:liya" → "liya"
    const persona = await loadPersonaFromFile(characterId);
    this.characterStore.getState().initCharacter(characterId, persona);
    // 从sessionFile恢复上一次的记忆快照（如果存在）
    const restored = await this.memory.restore(params.sessionFile);
    return { ok: true, restoredMessageCount: restored?.count ?? 0 };
  }

  // ─── 生命周期：写入 ──────────────────────────────────────

  async ingest(params: {
    sessionId: string;
    message: AgentMessage;
    isHeartbeat?: boolean;
  }): Promise<IngestResult> {
    if (params.isHeartbeat) return { ingested: false }; // 心跳消息不写入记忆
    const threadId = params.sessionId; // 直接用sessionId作为Mastra的threadId
    await this.memory.add({ threadId, content: params.message.content });
    return { ingested: true };
  }

  async ingestBatch(params: {
    sessionId: string;
    messages: AgentMessage[];
    isHeartbeat?: boolean;
  }): Promise<IngestBatchResult> {
    if (params.isHeartbeat) return { ingested: false, count: 0 };
    // 批量写入：一轮GOAP行动产生的多条事件一次性入库
    for (const msg of params.messages) {
      await this.memory.add({ threadId: params.sessionId, content: msg.content });
    }
    return { ingested: true, count: params.messages.length };
  }

  // ─── 生命周期：组装（核心循环步骤①）──────────────────────

  async assemble(params: {
    sessionId: string;
    messages: AgentMessage[];
    tokenBudget?: number;
  }): Promise<AssembleResult> {
    const characterId = this.parseCharacterId(params.sessionId);
    const persona = this.characterStore.getState().getPersona(characterId);
    const goals = this.characterStore.getState().getGoals(characterId);
    const shifts = this.characterStore.getState().getPersonaShifts(characterId);
    const worldState = getWorldState();

    // Mastra Semantic Recall：基于当前情境检索相关情节记忆
    const recalled = await this.memory.query({
      threadId: params.sessionId,
      query: worldState.currentSituation,
      topK: 5,
    });

    // 组装上下文，尊重tokenBudget
    const systemPrompt = this.buildSystemPrompt(persona, goals, shifts, recalled, worldState);
    const assembled: AgentMessage[] = [
      { role: 'system', content: systemPrompt },
      ...this.trimToTokenBudget(params.messages, params.tokenBudget),
    ];

    return {
      messages: assembled,
      estimatedTokens: estimateTokens(assembled),
      metadata: { recalledMemories: recalled, activeGoals: goals, personaShifts: shifts },
    };
  }

  // ─── 生命周期：压缩（Reflection入口）─────────────────────

  async compact(params: {
    sessionId: string;
    sessionFile: string;
    tokenBudget?: number;
    force?: boolean;
    currentTokenCount?: number;
    compactionTarget?: 'budget' | 'threshold';
    customInstructions?: string;
    runtimeContext?: NovelRuntimeContext;
  }): Promise<CompactResult> {
    const characterId = this.parseCharacterId(params.sessionId);

    // 判断是否需要压缩
    if (!params.force && !this.shouldCompact(params.currentTokenCount, params.tokenBudget)) {
      return { ok: true, compacted: false };
    }

    // 1. 利用Mastra Observational Memory自动压缩冗长对话
    const summary = await this.memory.compact({ threadId: params.sessionId });

    // 2. 触发Reflection Agent → 分析近期记忆 → 可能产生人格补丁
    const shift = await this.runReflection(characterId, params.customInstructions);

    // 3. 持久化session快照
    await this.memory.snapshot(params.sessionFile);

    return {
      ok: true,
      compacted: true,
      summary: summary?.text,
      personaShift: shift,
    };
  }

  // ─── 生命周期：轮次结束（核心循环步骤⑨）────────────────

  async afterTurn(params: {
    sessionId: string;
    sessionFile: string;
    messages: AgentMessage[];
    prePromptMessageCount: number;
    autoCompactionSummary?: string;
    isHeartbeat?: boolean;
    tokenBudget?: number;
    runtimeContext?: NovelRuntimeContext;
  }): Promise<void> {
    if (params.isHeartbeat) return;

    const characterId = this.parseCharacterId(params.sessionId);

    // 更新短期目标状态
    if (params.runtimeContext?.currentGoals) {
      this.characterStore.getState().updateGoals(characterId, params.runtimeContext.currentGoals);
    }

    // 如果有自动压缩摘要，写入记忆
    if (params.autoCompactionSummary) {
      await this.ingest({
        sessionId: params.sessionId,
        message: { role: 'system', content: `[auto-compact] ${params.autoCompactionSummary}` },
      });
    }

    // 评估是否触发compact（新增消息量/重大事件/token超预算）
    const newMsgCount = params.messages.length - params.prePromptMessageCount;
    if (newMsgCount > 20 || this.hasSignificantEvent(params.messages)) {
      await this.compact({
        sessionId: params.sessionId,
        sessionFile: params.sessionFile,
        tokenBudget: params.tokenBudget,
        runtimeContext: params.runtimeContext,
      });
    }
  }

  // ─── 生命周期：子Agent管理 ───────────────────────────────

  async prepareSubagentSpawn(params: {
    parentSessionKey: string;   // "novel:liya"
    childSessionKey: string;    // "novel:liya:director"
    ttlMs?: number;
  }): Promise<SubagentSpawnPreparation | undefined> {
    const childType = params.childSessionKey.split(':').pop(); // "director" | "reflection"

    // 为子Agent创建隔离的上下文切片
    await this.memory.fork({
      from: params.parentSessionKey,
      to: params.childSessionKey,
      filter: childType === 'director'
        ? { excludeSystemMessages: true }  // Director不需要认知推理的系统消息
        : undefined,
    });

    return {
      rollback: async () => {
        // 如果子Agent启动失败，清理fork的上下文
        await this.memory.deleteThread(params.childSessionKey);
      },
    };
  }

  async onSubagentEnded(params: {
    childSessionKey: string;    // "novel:liya:director"
    reason: SubagentEndReason;  // "completed" | "aborted" | "error" | "timeout"
  }): Promise<void> {
    const [, characterId, agentType] = params.childSessionKey.split(':');

    if (params.reason === 'completed') {
      if (agentType === 'director') {
        // Director输出的叙事文本 → 回写到角色主记忆流
        const output = await this.memory.getLatest(params.childSessionKey);
        await this.ingest({
          sessionId: `novel:${characterId}`,
          message: { role: 'assistant', content: output.content },
        });
      }
      if (agentType === 'reflection') {
        // Reflection的人格补丁 → 合并到Zustand Store
        const patch = await this.memory.getLatest(params.childSessionKey);
        this.characterStore.getState().applyPersonaShift(characterId, JSON.parse(patch.content));
      }
    }

    // 清理子Agent的临时上下文
    await this.memory.deleteThread(params.childSessionKey);
  }

  // ─── 生命周期：清理 ──────────────────────────────────────

  async dispose(): Promise<void> {
    await this.memory.close();
  }

  // ─── 私有方法 ────────────────────────────────────────────

  private parseCharacterId(sessionId: string): string {
    return sessionId.replace('novel:', '').split(':')[0];
  }

  private trimToTokenBudget(messages: AgentMessage[], budget?: number): AgentMessage[] {
    if (!budget) return messages;
    // 从最新消息往回保留，直到逼近budget
    // ...
    return messages;
  }

  private shouldCompact(current?: number, budget?: number): boolean {
    if (!current || !budget) return false;
    return current > budget * 0.85; // 85%阈值
  }
}

### 7.1 需要观测的指标

  const { phase, currentGoal, goapQueue, memoryContext, timeline, traces } = useDebugStore();
  const [collapsed, setCollapsed] = useState(false);

  if (collapsed) return <CollapseButton onClick={() => setCollapsed(false)} />;

  return (
    <aside className="debug-drawer">
      <PhaseIndicator phase={phase} />
      {currentGoal && <Goal5W1HPanel goal={currentGoal} />}
      <GoapQueuePanel queue={goapQueue} />
      <MemoryContextPanel context={memoryContext} />
      <TimelinePanel timeline={timeline} />
      <AgentTracePanel traces={traces} />
      <button onClick={() => setCollapsed(true)}>Collapse ◂</button>
    </aside>
  );
}
```

### 9.4 Drawer面板功能

| 面板 | 数据来源 | 交互能力 |
|------|---------|---------|
| **核心循环状态** | Workflow onStepFinish | 实时高亮当前阶段，显示耗时 |
| **5W1H结果** | Cognition Agent输出 | 可展开查看完整JSON，复制prompt |
| **GOAP行动队列** | GOAP Planner输出 | 逐个标记执行进度✅⏳○ |
| **记忆上下文** | ContextEngine.assemble的metadata | 展开查看每条recalled memory的相关度分数 |
| **时间线** | NQTR timeTracker + 宏观事件队列 | 标红冲突事件⚠，鼠标悬停预览 |
| **Agent Trace** | Mastra Trace Exporter | 显示每个Agent的延迟/token/tool调用 |
| **操控** | — | 暂停循环 / 单步执行 / 强制触发Reflection / 手动注入事件 |
